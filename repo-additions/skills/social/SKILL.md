# Social — Universal Social Listening & Brand Intelligence Skill

Monitors brand mentions, customer sentiment, competitor activity, and reputation
signals across free and accessible public platforms. Works for any business type.

---

## Quick Reference

| Command | What it does |
|--------|--------------|
| `/social <brand>` | **FULL MODE** — runs ALL sub-skills in parallel, unified Social report |
| `/social listen <brand>` | Scan all sources for brand mentions right now |
| `/social sentiment <brand>` | Aspect-based sentiment breakdown (staff, product, price, etc.) |
| `/social competitors <brand> vs <competitor>` | Side-by-side sentiment & presence comparison |
| `/social alerts <brand>` | Set up spike detection rules for sudden mention surges |
| `/social report <brand>` | Generate a weekly brand health summary report |
| `/social respond <review-text>` | Draft a professional response to a review or mention |

---

## Routing Logic

**If invoked as `/social <brand>` with only a brand name and no subcommand → run FULL MODE (see below).**

Otherwise detect the subcommand and route accordingly:
- `/social listen` → `social-listen`
- `/social sentiment` → `social-sentiment`
- `/social competitors` → `social-competitors`
- `/social alerts` → `social-alerts`
- `/social report` → `social-report`
- `/social respond` → `social-respond`
- `/social` with no arguments → show quick reference table above

## FULL MODE — Run All Social Sub-Skills at Once

When invoked as `/social <brand>` (just a brand name, no subcommand):

### Step 1 — Detect Business Type
Infer the business type from the brand name, context, or any URL provided:

| Business type | Key aspects to track |
|---------------|---------------------|
| Hotel / Hospitality | Cleanliness, staff, location, food, value |
| Restaurant / Food | Food quality, service, wait time, atmosphere, price |
| SaaS / Software | Onboarding, support, pricing, features, bugs |
| Retail / E-commerce | Shipping, product quality, returns, customer service |
| Healthcare | Staff, wait time, cleanliness, billing, care quality |
| Agency / B2B | Communication, deliverables, results, pricing |
| Local business | Location, hours, staff, pricing, parking |

### Step 2 — Run All Sub-Skills in Parallel

Launch ALL of the following simultaneously:

**2a — Mention Scanning** (`social-listen`)
- Scan Reddit, YouTube, Yelp, Google News, Web/Forums for brand mentions
- Total mention count and recency
- Dominant sentiment per platform
- Any crisis signals or viral complaints

**2b — Sentiment Analysis** (`social-sentiment`)
- Overall Sentiment Score (0–100) + band
- Aspect-by-aspect breakdown (adapted to business type)
- Top 3 praised aspects with direct quotes
- Top 3 complained-about aspects with direct quotes
- Data confidence level

**2c — Competitor Comparison** (`social-competitors`)
- Social Health Score for brand + top 3 competitors
- Side-by-side comparison: sentiment, volume, response rate, platform coverage
- Where competitors are winning the conversation
- Where the target brand has an advantage
- Platforms where competitors are active but target is missing

**2d — Brand Health Report** (`social-report`)
- Social Health Score (0–100) + band
- Mention volume by platform
- Sentiment trend: improving / stable / declining
- Top 5 themes (positive + negative) with quotes and sources
- Active alerts: crisis / urgent / watch

**2e — Alert Configuration** (`social-alerts`)
- Recommended spike detection thresholds for this brand
- Crisis signal definitions (what triggers an alert)
- Suggested monitoring schedule

## Scoring Model — Social Health Index

Every brand analysis produces a **Social Health Score (0–100)**:

| Dimension | Weight |
|-----------|--------|
| Sentiment Ratio (positive vs negative) | 35% |
| Review Volume & Recency | 20% |
| Response Rate (brand replies to reviews) | 20% |
| Mention Trend (growing vs shrinking) | 15% |
| Competitive Standing | 10% |

**Bands:**
- 90–100: Excellent
- 75–89: Good
- 60–74: Fair
- 40–59: Poor
- <40: Critical — immediate reputation action needed

---

## Global Rules

- Only use **publicly accessible** data — no scraping behind logins
- Never fabricate reviews, scores, or sentiment counts
- If a source is inaccessible, note it and skip — do not guess
- Always state data recency (when the data was last fetched or estimated)
- Recommendations must be **actionable and specific** — not generic advice
- Match tone of response drafts to the brand's industry and voice
