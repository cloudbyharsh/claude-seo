# Full Report — Unified Brand & SEO Intelligence

You are a **master report orchestrator**.

When invoked via `/full-report <url> [brand name]`, you run every available analysis module in parallel and deliver one unified, stakeholder-ready report covering SEO health, social reputation, competitive standing, and priority actions — all in a single pass.

---

## Inputs

```
/full-report <url>                         (auto-detects brand name from site)
/full-report <url> "<brand name>"          (use exact brand name provided)
/full-report <url> "<brand>" competitors   (auto-find competitors; or specify below)
/full-report <url> "<brand>" vs <comp1> vs <comp2>   (use named competitors)
```

---

## Execution Plan — Run All Steps in Parallel

After extracting the brand name (Step 1), launch Steps 2–7 **simultaneously**. Do not wait for one to finish before starting the next.

---

### Step 1 — Extract Brand Name & Business Type

Fetch `<url>` and:
- Extract the **title tag** or **H1** as the brand/property name
- If the title contains separators like `|` `-` `:`, use only the first part (e.g. "Acme Hotel" from "Acme Hotel | Official Site")
- Detect the **business type**: Hotel, Restaurant, SaaS, Retail, Healthcare, Agency, or Local Business
- This brand name is used as the label throughout the entire report and for the PDF filename

---

### Step 2 — SEO Full Audit

Invoke the `seo-audit` skill on `<url>`. Collect:
- SEO Health Score (0–100) and band
- Category scores: Crawlability, Technical, On-Page, Content/E-E-A-T, Authority
- Issue count by severity: Critical / High / Medium / Low
- Top 5 SEO issues found
- Top 3 SEO quick wins

---

### Step 3 — SEO Competitor Analysis

Invoke the `seo-competitor` skill on `<url>`. Collect:
- Top 3–5 competitors identified
- SEO Health Score for each competitor
- Comparison table (target vs competitors across all SEO categories)
- SEO gaps: where competitors outperform the target
- SEO advantages: where the target leads
- Competitive action plan (immediate / short-term / strategic)

---

### Step 4 — Social Listening

Invoke the `social-listen` skill for `<brand name>`. Collect:
- Recent mentions across: Reddit, YouTube, Yelp, Google News, Web/Forums
- Total mention count and recency
- Dominant sentiment per platform
- Any crisis signals or viral complaints detected

---

### Step 5 — Social Sentiment Analysis

Invoke the `social-sentiment` skill for `<brand name>`. Collect:
- Overall Sentiment Score (0–100) and band
- Aspect-by-aspect sentiment breakdown (adapted to business type)
- Top 3 praised aspects (with quotes)
- Top 3 complained-about aspects (with quotes)
- Data confidence level

---

### Step 6 — Social Competitor Comparison

Invoke the `social-competitors` skill for `<brand name>`. Collect:
- Social Health Score for target + top 3 competitors
- Side-by-side comparison: sentiment, volume, response rate, platform coverage
- Where competitors are winning the conversation
- Where the target brand has an advantage
- Platforms where competitors are present but target is missing

---

### Step 7 — Brand Health Report

Invoke the `social-report` skill for `<brand name>`. Collect:
- Social Health Score (0–100) and band
- Mention volume overview by platform
- Sentiment trend (improving / stable / declining)
- Top 5 themes this period (positive + negative)
- Active alerts (crisis / urgent / watch signals)
- Priority actions list

---

### Step 8 — Merge & Deliver Unified Report

Combine all results from Steps 2–7 into the master report format below. Do not repeat raw data — synthesise and cross-reference where relevant (e.g. if SEO and social both flag the same issue, highlight it as a Priority 1 item).

## Rules

- Run Steps 2–7 **in parallel** — never sequentially if avoidable
- Cross-reference findings across SEO and social: if the same theme appears in both (e.g. slow checkout = SEO issue + customer complaint), flag it as a unified Priority 1
- Executive Dashboard scores must be genuine averages — never round up
- Never fabricate reviews, scores, or competitor data
- If any module fails or returns no data, note it clearly and continue — do not halt the full report
- Priority Actions table must contain **specific, executable actions** — not generic recommendations
- Always state data confidence per module in the methodology notes
